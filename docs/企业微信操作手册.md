# 项目经理数字员工系统 - 飞书操作手册

## 一、系统概述

### 1.1 系统简介

项目经理数字员工系统是一个基于飞书的智能项目管理助手，通过自然语言交互帮助项目经理完成日常事务性工作。

**唯一交互入口：飞书**

### 1.2 当前服务状态

| 服务名称 | 容器名称 | 状态 | 端口 |
|----------|----------|------|------|
| 主应用服务 | pm_app | ✅ 运行中 | 28000 |
| PostgreSQL数据库 | pm_postgres | ✅ 运行中 | 15432 |
| Redis缓存 | pm_redis | ✅ 运行中 | 16379 |
| RabbitMQ消息队列 | pm_rabbitmq | ✅ 运行中 | 15672/25672 |
| Celery异步任务 | pm_celery_worker | ⚠️ 异常 | - |

> **注意：** Celery Worker 存在配置问题，正在持续重启。这不影响主要功能使用，但异步任务（如周报生成）可能暂时不可用。

### 1.3 服务管理命令

```bash
# 进入项目目录
cd /root/data/disk/projects/pm-digital-employee

# 启动服务
./manage.sh start

# 停止服务
./manage.sh stop

# 重启服务
./manage.sh restart

# 查看状态
./manage.sh status

# 查看日志
./manage.sh logs app

# 重新构建
./manage.sh build
```

## 二、核心功能列表

### 2.1 一期MVP功能（10项）

| 序号 | 功能名称 | 功能描述 | 使用方式 |
|------|----------|----------|----------|
| 1 | **项目总览查询** | 查询项目整体状态、进度、里程碑、风险、成本 | "查询XX项目概况" |
| 2 | **项目周报生成** | 自动汇总本周任务进展、下周计划、风险状态 | "生成XX项目周报" |
| 3 | **WBS自动生成** | 根据项目信息生成工作分解结构 | "为XX项目生成WBS" |
| 4 | **任务进度更新** | 更新任务进度、状态、备注 | "更新XX任务进度为80%" |
| 5 | **风险识别与预警** | 识别项目风险、发出预警提示 | "分析XX项目风险" |
| 6 | **成本监控** | 对比预算与实际支出、预警超支风险 | "查询XX项目成本情况" |
| 7 | **制度规范答疑** | 基于RAG回答项目管理规章制度问题 | "项目立项流程是什么" |
| 8 | **项目情况咨询** | 回答项目具体情况相关问题 | "XX项目的里程碑有哪些" |
| 9 | **会议纪要生成** | 根据会议内容生成结构化纪要 | "帮我生成会议纪要" |
| 10 | **预立项/立项材料合规初审** | 审核材料合规性 | "审核这份立项材料" |

## 四、系统架构说明

### 4.1 九层架构

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 接入层        │ 飞书 Bot Webhook、Callback、File Upload      │
├─────────────────────────────────────────────────────────────────┤
│ 2. API网关层     │ FastAPI、验签、幂等、trace_id、异常处理       │
├─────────────────────────────────────────────────────────────────┤
│ 3. 会话上下文层  │ 用户/群/项目上下文、对话状态机、多轮补参     │
├─────────────────────────────────────────────────────────────────┤
│ 4. 权限隔离层    │ 用户-项目权限、群-项目绑定、Skill权限校验    │
├─────────────────────────────────────────────────────────────────┤
│ 5. 编排层        │ Intent Router、Skill Registry、Orchestrator  │
├─────────────────────────────────────────────────────────────────┤
│ 6. AI能力层      │ LLM Gateway、Prompt Manager、RAG、安全防护   │
├─────────────────────────────────────────────────────────────────┤
│ 7. Skill插件层   │ 10个核心Skill，BaseSkill规范，manifest      │
├─────────────────────────────────────────────────────────────────┤
│ 8. 集成适配层    │ 项目管理、财务、DevOps、OA、飞书适配器      │
├─────────────────────────────────────────────────────────────────┤
│ 9. 数据层        │ PostgreSQL、Redis、pgvector、Celery         │
└─────────────────────────────────────────────────────────────────┘
```

## 六、故障排查指南

### 6.1 常见问题

#### 问题1：飞书消息无响应

**排查步骤：**
```bash
# 1. 检查服务状态
./manage.sh status

# 2. 检查日志
./manage.sh logs app | grep -i error

# 3. 检查飞书配置
cat .env | grep LARK
```

#### 问题2：Celery Worker 异常

**当前状态：** Celery Worker 存在配置问题，正在持续重启

**影响范围：** 异步任务（周报生成、批量处理等）可能不可用

**临时方案：** 主功能正常，同步操作可用

#### 问题3：数据库连接失败

```bash
# 检查数据库状态
docker exec pm_postgres pg_isready

# 重启数据库
docker-compose restart postgres
```

### 6.2 日志查看

```bash
# 实时查看应用日志
docker-compose logs -f app

# 查看最近100行日志
docker-compose logs --tail=100 app

# 搜索错误日志
docker-compose logs app | grep -i error
```
