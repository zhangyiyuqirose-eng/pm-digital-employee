# 全文档智能解析与自动录入功能需求规格说明书

**版本**: V3.0  
**日期**: 2026-04-19  
**项目**: PM数字员工系统 v1.3.0

## 一、概述

### 1.1 需求背景

现有PM数字员工系统仅支持三种数据录入方式：
1. **飞书卡片录入**：用户通过交互式卡片单条录入数据
2. **Excel模板导入**：用户填写标准模板后批量导入
3. **飞书在线表格同步**：与飞书表格双向实时同步

这些方式均需人工整理结构化数据，操作成本高、效率低。项目全生命周期产生大量非结构化文档（周报、会议纪要、WBS、风险登记表等），目前无法自动处理，数据录入耗时耗力。

### 1.2 需求目标

为PM数字员工新增**任意格式项目文档智能解析与自动入库**能力，实现：

- 用户在飞书单聊或群聊直接发送文档，无需额外操作
- 系统自动完成文档分类→内容识别→信息提取→结构化转换→自动入库全流程
- 提取的信息精准录入到系统对应的skill数据库中
- 所有项目成员可通过自然语言查询或系统功能模块调用

### 1.3 系统定位

本功能作为**第四种数据录入通道**，融入现有9层架构：

| 层级 | 新增内容 |
|------|----------|
| 接入层 | 飞书文件消息处理（新增file/image消息类型） |
| 编排层 | DocumentParseSkill（新Skill） |
| AI能力层 | 文档分类、信息提取、置信度评估Prompt |
| Skill插件层 | DocumentParseSkill + 数据入库调用 |
| 数据层 | DocumentParseRecord（新增模型） |

## 三、功能详细需求

### 3.1 自动分类能力

#### 3.1.1 分类维度

| 维度 | 分类依据 | 输出 |
|------|----------|------|
| 文档大类 | 内容特征分析 | project_doc/management_doc/external_doc/other |
| 项目阶段 | 文档内容关键词 | pre_initiation/initiation/execution/closing/full_cycle |
| 文档子类型 | 文件名+内容匹配 | weekly_report/meeting_minutes/wbs/risk_register等 |
| 关联项目 | 项目名称/编号匹配 | project_id |

#### 3.1.2 分类流程

```
文件扩展名 → 文件名关键词 → 内容特征分析(LLM) → 项目关联推断 → 综合判定
```

#### 3.1.3 分类结果修正

- 分类置信度<80%时发送澄清卡片
- 用户修正结果自动学习优化后续分类
- 修正记录存储用于模型迭代

### 3.2 智能识别与信息提取能力

#### 3.2.1 提取实体与字段

| 实体类型 | 提取字段 | 文档来源 |
|----------|----------|----------|
| Project | name, code, status, start_date, end_date, total_budget, pm_name | 立项材料 |
| Task | name, status, priority, start_date, end_date, assignee_name, progress | WBS、任务清单、会议纪要 |
| Milestone | name, status, due_date, is_key_milestone | 里程碑计划 |
| Risk | title, category, level, probability, impact, owner_name, mitigation_plan | 风险登记表 |
| Cost | category, amount, expense_date | 成本核算表 |
| WeeklyReport | report_date, week_start, week_end, summary, completed_tasks, next_week_plan | 周报文档 |
| MeetingMinutes | meeting_title, meeting_date, attendees, content, action_items | 会议纪要 |
| WBSVersion | wbs_data (JSON树形结构) | WBS文档 |

#### 3.2.2 字段提取规则

| 字段类型 | 规则 |
|----------|------|
| 日期 | 转换为YYYY-MM-DD格式，支持多种输入格式 |
| 金额 | 提取数字，默认单位元，支持万元转换 |
| 状态 | 映射为标准枚举值（见附录A） |
| 人员 | 提取姓名，系统自动匹配飞书用户ID |
| 进度 | 提取百分比，去除%符号 |
| 优先级 | 映射为low/medium/high/critical |

#### 3.2.3 置信度机制

| 置信度范围 | 处理策略 |
|------------|----------|
| >=95% | 自动入库，发送成功通知 |
| 80%-95% | 自动入库，发送入库确认通知 |
| 60%-80% | 发送待确认卡片，用户确认后入库 |
| <60% | 标记为待处理，需人工审核 |

### 3.3 自动入库能力

#### 3.3.1 入库流程

```
数据校验 → 项目关联校验 → 权限检查 → 数据入库 → 结果反馈 → 版本记录
```

#### 3.3.2 数据校验

调用现有ValidationService，校验规则：
- 必填字段校验
- 数据类型校验
- 业务规则校验（如结束日期>=开始日期）
- 范围校验（如进度0-100）

#### 3.3.3 数据来源标识

入库数据标记data_source为`document_parse`，并记录：
- external_id: 文档解析记录ID
- sync_version: 版本号
- last_sync_at: 入库时间

#### 3.3.4 操作日志

所有入库操作记录：
- 文档来源（file_key）
- 提取时间
- 提取内容
- 操作人员
- 入库结果

### 3.4 异常处理能力

#### 3.4.1 异常类型与处理

| 异常类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 文件下载失败 | API超时/文件过期 | 重试3次，失败发错误卡片 |
| 文件解析失败 | 格式不支持/文件损坏 | 发格式不支持提示 |
| OCR识别失败 | 图片模糊 | 发识别失败提示 |
| 分类失败 | 无法识别类型 | 发澄清卡片 |
| 置信度低 | <80% | 发待确认卡片 |
| 数据冲突 | 新旧数据不一致 | 发冲突解决卡片 |
| 权限不足 | 无写入权限 | 发权限提示 |
| 项目未关联 | 无法推断项目 | 发项目选择卡片 |

#### 3.4.2 数据冲突解决策略

| 策略 | 适用场景 |
|------|----------|
| MANUAL | 置信度<80%，需人工确认 |
| LAST_WRITE | 置信度>=80%，时间戳最新优先 |
| SOURCE_PRIORITY | document_parse优先级高于其他来源 |
| USER_DECISION | 用户主动选择采用新/旧数据 |

## 四、交互需求

### 4.1 单聊文档发送流程

```
用户发送文档 → 系统接收 → 发送"正在处理"提示 → 分类/提取/入库 
→ [置信度>=80%] 发送成功卡片
→ [置信度<80%] 发送待确认卡片 → 用户确认 → 入库 → 发送结果
```

### 4.2 群聊文档发送流程

```
用户发送文档 → 系统接收 → 检查群绑定项目 → 权限校验 
→ [无权限] 发送权限提示
→ [有权限] 执行处理 → 发送群可见结果卡片
```

### 4.3 处理结果反馈

| 场景 | 卡片类型 | 内容 |
|------|----------|------|
| 成功入库 | 成功卡片 | 文档名、分类、置信度、入库记录数 |
| 待确认 | 确认卡片 | 数据预览、缺失字段、确认按钮 |
| 冲突 | 冲突卡片 | 新旧数据对比、选择按钮 |
| 失败 | 错误卡片 | 错误类型、建议操作 |

### 4.4 待确认信息审核

- 提供"待确认信息"列表入口
- 用户可查看所有待确认文档
- 支持一键审核和修正
- 修正后重新触发入库

## 五、权限与安全需求

### 5.1 权限控制

#### 5.1.1 文档处理权限

- 用户必须是项目成员（有UserProjectRole记录）
- 用户角色必须有数据写入权限
- 群聊场景需检查群绑定项目一致性

#### 5.1.2 项目关联权限

| 场景 | 规则 |
|------|------|
| 群聊 | 项目必须与群绑定一致 |
| 单聊 | 用户必须为项目成员 |
| 无法推断 | 需用户选择有权限的项目 |

### 5.2 安全要求

#### 5.2.1 文件安全

- 文件类型白名单限制
- 文件大小限制（50MB）
- 禁止上传可执行文件
- 内容安全检查（SafetyGuard）

#### 5.2.2 数据安全

- 敏感信息脱敏（金额显示万元级别）
- 原始文档加密存储
- 操作审计日志完整记录
- 数据版本控制，支持回滚

#### 5.2.3 Prompt安全

- 文档内容经过SafetyGuard检测
- 防止文档内容包含注入指令
- 输出内容安全检查
