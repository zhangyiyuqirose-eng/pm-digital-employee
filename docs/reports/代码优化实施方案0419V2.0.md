## 一、方案概述

### 1.1 优化目标

将参考版本v1.2.0的多源数据录入核心功能移植到本地版本，补充缺失的服务层和API层，同时保持本地版本的架构优势。

### 1.2 优化原则

| 原则 | 说明 |
|------|------|
| **最小侵入** | 不修改本地现有核心模块（Orchestrator、Skills、权限层） |
| **架构兼容** | 适配本地Repository框架、依赖注入、异常体系 |
| **渐进实施** | 按依赖顺序分阶段实施，每阶段可独立验证 |
| **测试先行** | 每个新增模块同步编写单元测试 |

### 1.3 实施范围

| 类别 | 操作 | 文件数 |
|------|------|--------|
| 新增服务 | 移植4个核心服务 | 4 |
| 新增API | 移植2个API端点 | 2 |
| 新增配置 | 移植校验配置+枚举补充 | 2 |
| 新增测试 | 编写单元测试 | 5 |
| 适配调整 | 代码适配修改 | ~10处 |
| **总计** | | **15+文件** |

本地版本适配方案A（推荐）：实现`sheets`属性代理
```python
# 在 app/integrations/lark/client.py 中添加

class LarkSheetsProxy:
    """飞书表格API代理."""
    
    def __init__(self, client: LarkClient):
        self._client = client
    
    async def read_cells(self, spreadsheet_token: str, sheet_id: str, range: str) -> Dict:
        """读取表格单元格."""
        return await self._client.get(
            f"/sheets/v2/spreadsheets/{spreadsheet_token}/values/{sheet_id}",
            params={"range": range}
        )
    
    async def write_cells(self, spreadsheet_token: str, sheet_id: str, range: str, values: List) -> Dict:
        """写入表格单元格."""
        return await self._client.put(
            f"/sheets/v2/spreadsheets/{spreadsheet_token}/values",
            json_data={"sheetId": sheet_id, "range": range, "values": values}
        )

# 在 LarkClient 类中添加属性
class LarkClient:
    ...
    @property
    def sheets(self) -> LarkSheetsProxy:
        return LarkSheetsProxy(self)
```

"""
ValidationService 单元测试

import pytest
from app.services.validation_service import ValidationService

class TestValidationService:
    """ValidationService测试类."""
    
    @pytest.fixture
    def service(self):
        return ValidationService()
    
    def test_validate_required_fields_missing(self, service):
        """测试必填字段缺失."""
        data = {"name": ""}  # name为空
        result = service.validate_all(data, "project")
        assert result.is_valid == False
        assert any(e["error_type"] == "required" for e in result.errors)
    
    def test_validate_data_type_conversion(self, service):
        """测试数据类型转换."""
        data = {
            "name": "测试项目",
            "project_type": "研发项目",
            "total_budget": "10000.50",  # 字符串转float
        }
        result = service.validate_all(data, "project")
        assert result.is_valid == True
        assert result.validated_data["total_budget"] == 10000.50
    
    def test_validate_business_rule_date_compare(self, service):
        """测试业务规则日期比较."""
        data = {
            "name": "测试项目",
            "project_type": "研发项目",
            "start_date": "2026-04-20",
            "end_date": "2026-04-10",  # 结束日期早于开始日期
        }
        result = service.validate_all(data, "project")
        assert result.is_valid == False
        assert any(e["error_type"] == "business" for e in result.errors)
    
    def test_validate_batch_success(self, service):
        """测试批量校验."""
        data_list = [

            {"name": "项目2", "project_type": "运维项目"},
        ]
        results = service.validate_batch(data_list, "project")
        assert len(results) == 2
        assert all(r.is_valid for r in results)
