## 一、交付内容概览

### 1.1 新增功能概述

本功能为 PM Digital Employee 系统新增第 4 种数据录入渠道，实现用户通过飞书发送项目管理文档，系统自动解析、分类、提取数据并录入数据库的能力。

**核心特性**:
- 支持 12 种文档格式：DOCX、PDF、XLSX、XLS、CSV、PPTX、TXT、MD、JPG、PNG、BMP
- 智能文档分类：自动识别周报、会议纪要、WBS、风险登记表等 7 大类文档
- LLM 信息提取：基于 Prompt 模板的智能数据提取
- 置信度分级处理：>=95% 自动入库、>=80% 需确认、>=60% 需补参
- 用户确认机制：飞书卡片交互确认和修正
- 完整异常处理：失败重试、错误分类、用户反馈

## 三、核心组件实现详情

### 3.1 FileParserService (文件解析服务)

**功能**: 解析多种格式文档，提取文本内容、表格数据、结构化信息

**支持格式**:
- `DOCX`: python-docx 解析，提取段落、表格
- `PDF`: pdfplumber 解析，OCR placeholder
- `XLSX/XLS`: openpyxl 解析，多sheet处理
- `CSV`: csv 模块解析
- `PPTX`: python-pptx 解析
- `TXT/MD`: 直接读取文本
- `JPG/PNG/BMP`: OCR placeholder (Tesseract)

**核心类**:
```python
class FileInfo:
    file_key: str
    file_name: str
    file_type: str
    file_size: int
    storage_path: Optional[str]

class ParsedContent:
    text_content: str
    tables: List[TableData]
    metadata: Dict[str, Any]
    has_images: bool

class FileParserService:
    async def parse_file(self, file_path: str) -> ParsedContent
    async def get_file_info(self, file_path: str) -> FileInfo
```

### 3.2 DocumentClassifierService (文档分类服务)

**功能**: 智能识别文档类型和项目关联

**分类策略** (5级判定):
1. **扩展名判定**: 特定格式对应特定类型 (如 .xlsx → 表格类)
2. **文件名关键词**: 关键词匹配 (如包含"周报" → weekly_report)
3. **内容特征**: 内容结构分析
4. **LLM分类**: Prompt模板智能分类
5. **项目匹配**: 从文档内容推断关联项目

**文档大类**:
- `weekly_report`: 项目周报
- `meeting_minutes`: 会议纪要
- `wbs`: WBS工作分解结构
- `risk_register`: 风险登记表
- `milestone_plan`: 里程碑计划
- `cost_report`: 成本报告
- `other`: 其他文档

**核心类**:
```python
class ClassificationResult:
    document_category: DocumentCategory
    document_subtype: Optional[str]
    project_phase: Optional[str]
    confidence: float
    inferred_entity_types: List[str]
    project_match: Optional[ProjectMatch]
```

### 3.3 DataExtractorService (数据提取服务)

**功能**: 从文档内容提取结构化数据

**实体类型定义** (9种):
```python
ENTITY_SCHEMAS = {
    "Task": {
        "fields": ["task_name", "assignee_name", "status", "progress", ...],
        "required": ["task_name"],
        "enums": {"status": STATUS_MAPPING}
    },
    "Milestone": {...},
    "Risk": {...},
    "Cost": {...},
    "WeeklyReport": {...},
    "MeetingMinutes": {...},
    "WBSItem": {...},
    "RiskItem": {...},
    "ActionItem": {...}
}
```

**置信度评估**:
- 字段级置信度: 每个提取字段的置信度
- 实体级置信度: 整体提取质量
- 缺失字段标记: 记录未提取到的必填字段

### 3.4 DataImportService (数据导入服务)

**功能**: 将提取数据导入领域实体数据库

**处理流程**:
1. 调用 ValidationService 校验数据格式
2. 调用 SyncEngine 处理数据同步
3. 记录导入结果和冲突信息

**集成复用**:
- ValidationService: 统一校验规则
- SyncEngine: 数据同步引擎
- DataSource.document_parse: 来源标识

### 3.5 DocumentParseService (主编排服务)

**功能**: 协调完整解析流程

**处理流程**:
```
文件下载 → 格式解析 → 文档分类 → 数据提取 → 数据导入 → 用户反馈
     │           │           │           │           │
     ▼           ▼           ▼           ▼           ▼
  Lark API   FileParser  Classifier   Extractor   Importer
```

**置信度分级处理**:
```python
if confidence >= 0.95:
    # 自动入库，发送成功卡片
elif confidence >= 0.80:
    # 需用户确认，发送确认卡片
elif confidence >= 0.60:
    # 需补参，发送补参卡片
else:
    # 失败，发送错误卡片
```

### 3.6 DocumentParseRecord (ORM模型)

**功能**: 记录完整解析过程和结果

**字段分组**:
- 文件信息: file_key, file_name, file_type, file_size, storage_path
- 分类结果: document_category, document_subtype, project_phase, confidence
- 项目关联: inferred_project_id, confirmed_project_id, project_match_type
- 提取结果: extracted_data, entity_types, field_confidences, missing_fields
- 处理状态: parse_status, import_status, imported_entity_ids, conflict_ids
- 用户确认: requires_confirmation, confirmed_by, confirmed_at, confirmation_action
- 处理元数据: parser_version, llm_model, llm_tokens, processing_time_ms
- 异常信息: error_type, error_message, retry_count
- 来源信息: sender_id, sender_name, chat_id, chat_type, message_id

**数据库索引**:
- `ix_document_parse_records_file_key`: 文件Key索引
- `ix_document_parse_records_inferred_project_id`: 项目关联索引
- `ix_document_parse_records_parse_status`: 解析状态索引
- `ix_document_parse_records_sender_id`: 发送者索引
- `ix_document_parse_records_created_at`: 时间索引
- `ix_document_parse_records_requires_confirmation`: 确认需求索引

## 四、Skill插件实现

### 4.1 DocumentParseSkill

**功能**: 处理文件解析请求

**执行流程**:
1. 获取文件信息参数
2. 调用 DocumentParseService.parse_and_import()
3. 根据结果构建飞书卡片响应

**卡片类型**:
- **成功卡片**: 自动入库成功，展示提取数据摘要
- **确认卡片**: 需用户确认，展示提取数据供修正
- **补参卡片**: 需补充信息，展示缺失字段
- **错误卡片**: 解析失败，展示错误原因和重试选项

### 4.2 DocumentConfirmSkill

**功能**: 处理用户确认操作

**确认动作**:
- `confirm_import`: 确认导入数据
- `reject_import`: 拒绝导入
- `modify_data`: 修改数据后导入
- `cancel`: 取消操作

**执行流程**:
1. 解析卡片回调数据
2. 获取解析记录
3. 根据用户动作更新状态
4. 执行数据导入或取消

### 6.3 常见问题处理

**Q1: 文件发送后无响应？**
- 检查文件格式是否支持
- 确认文件大小 <= 50MB
- 等待处理完成 (大文件可能需30秒+)

**Q2: 解析置信度较低？**
- 文档格式不规范，建议使用标准模板
- 文档内容不完整，补充缺失信息
- 文档类型不明确，建议在文件名中标注

**Q3: 提取数据有误？**
- 点击"修改数据"进行修正
- 在确认卡片中直接编辑字段
- 反馈问题帮助改进提取规则
