## 二、核心功能实现

### 2.1 文件解析服务

支持格式：
- 文本文档：DOCX、PDF、TXT、MD
- 表格文档：XLSX、XLS、CSV
- 演示文档：PPTX
- 图片文档：JPG、PNG、BMP（OCR）

关键特性：
- 文件大小限制：50MB
- 格式白名单检查
- 表格数据提取
- OCR服务接口（待对接）

### 2.2 文档分类服务

分类维度：
- 文档大类：project_doc / management_doc / external_doc / other
- 项目阶段：pre_initiation / initiation / execution / closing / full_cycle
- 文档子类型：weekly_report / meeting_minutes / wbs / risk_register等

分类策略：
1. 扩展名快速分类（置信度0.5）
2. 文件名关键词匹配（置信度0.85）
3. 内容关键词分析（置信度0.7）
4. LLM深度分析（置信度动态）
5. 项目关联推断

### 2.3 数据提取服务

支持实体类型：
- Project、Task、Milestone、Risk、Cost
- WeeklyReport、MeetingMinutes、WBSVersion

关键特性：
- 字段置信度计算
- 状态值映射（中文→英文）
- 日期格式转换
- 人员姓名提取

### 2.4 数据入库服务

入库流程：
1. 数据校验（ValidationService）
2. 权限检查
3. 冲突检测
4. 实体创建
5. 数据来源标记（DataSource.DOCUMENT_PARSE）

### 2.5 主协调服务

完整处理流程：
1. 文件下载
2. 文件解析
3. 文档分类
4. 信息提取
5. 置信度评估
6. 数据入库（>=80%置信度）
7. 结果反馈

置信度阈值：
- >=95%：自动入库，发送成功通知
- >=80%：自动入库，发送确认通知
- >=60%：发送待确认卡片
- <60%：需人工审核

## 三、飞书集成扩展

### 3.1 Webhook扩展

新增消息类型处理：
- `file`：文件消息
- `image`：图片消息
- `media`：媒体消息

处理流程：
1. 接收消息 → 幂等检查
2. 发送处理提示
3. 异步调用DocumentParseSkill
4. 发送结果卡片

### 3.2 卡片设计

成功卡片：
- 文档名、分类、置信度、入库记录数

待确认卡片：

- 确认入库/修改数据/取消按钮

冲突卡片：
- 新旧数据对比、差异字段
