### 3.2 Validation Service 详细对比

#### 参考版本实现亮点（validation_service.py - 887行）

**校验流程架构**:

```
┌─────────────────────────────────────────────────────────────┐
│                     validate_all() 统一入口                  │
├─────────────────────────────────────────────────────────────┤
│  1. validate_required_fields() → 必填字段校验               │
│  2. validate_data_types() → 数据类型+范围+枚举校验           │
│  3. validate_business_rules() → 业务规则校验                │
├─────────────────────────────────────────────────────────────┤
│  ValidationResult 返回                                      │
│  - is_valid: bool                                           │
│  - errors: List[FieldError]                                 │
│  - warnings: List[Dict]                                     │
│  - validated_data: Dict (转换后的数据)                       │
└─────────────────────────────────────────────────────────────┘
```

**核心技术亮点**:

1. **类型转换+校验一体化**:
   ```python
   def _check_and_convert_type(value, expected_type, field_name):
       if expected_type == "date":
           # 支持多种日期格式
           for fmt in ["%Y-%m-%d", "%Y/%m/%d", "%Y%m%d"]:
               try:
                   return True, datetime.strptime(value, fmt).date()
               except ValueError:
                   continue
       elif expected_type == "bool":
           # 中文布尔值支持
           if value.lower() in ["true", "是", "yes", "1"]:
               return True, True
   ```

2. **业务规则校验**:
   - `compare`: 字段比较规则（结束日期 >= 开始日期）
   - `range`: 范围规则（金额 >= 0）
   - `custom`: 自定义规则预留扩展

3. **批量校验优化**:
   ```python
   def validate_batch(data_list, module) -> List[ValidationResult]:
       results = []
       for idx, data in enumerate(data_list):
           result = self.validate_all(data, module)
           if not result.is_valid:
               for error in result.errors:
                   error["row_index"] = idx + 1
           results.append(result)
       return results
   ```

4. **校验摘要统计**:
   ```python
   def get_validation_summary(results) -> Dict:
       return {
           "success_rate": valid_count / total,
           "error_type_counts": {...},  # 按错误类型统计
           "error_field_counts": {...},  # 按字段统计
       }
   ```

## 附录C: 主方案设计要点摘要

**三种导入模式**:
- `full_replace`: 全量替换
- `incremental_update`: 增量更新
- `append_only`: 仅追加

**四种同步方向**:
- `to_sheet`: 数据库→表格
- `from_sheet`: 表格→数据库
- `bidirectional`: 双向
- `realtime`: 实时同步

**冲突解决策略**:
- `LAST_WRITE`: 最后写入优先
- `SOURCE_A/B`: 来源优先
- `MANUAL`: 人工解决
- `MERGE`: 合并

**五模块支持**:
- `project` | `task` | `milestone` | `risk` | `cost`
