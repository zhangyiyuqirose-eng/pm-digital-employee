## 1. 飞书卡片录入操作说明

### 1.1 功能概述

飞书卡片录入是通过飞书消息中的交互式卡片，便捷地录入项目数据的方式。适合快速录入单条数据或少量数据。

### 1.2 适用场景

- 快速录入单个项目信息
- 快速录入单个任务信息
- 快速录入风险、里程碑、成本等数据
- 在移动端飞书中录入数据

### 1.3 操作步骤

#### 步骤1：触发录入卡片

在与PM数字员工对话中，输入以下指令：

- `录入项目` - 打开项目录入卡片
- `录入任务` - 打开任务录入卡片
- `录入风险` - 打开风险录入卡片
- `录入里程碑` - 打开里程碑录入卡片

#### 步骤2：填写卡片表单

系统将发送一张交互式卡片，包含表单字段：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 项目名称 | 项目的名称 | 是 |
| 项目编号 | 项目唯一标识 | 是 |
| 开始日期 | 项目开始时间 | 是 |
| 结束日期 | 项目结束时间 | 是 |
| 项目经理 | 项目负责人 | 是 |
| 预算金额 | 项目预算 | 否 |
| 项目状态 | 当前状态（下拉选择） | 是 |

#### 步骤3：提交数据

点击卡片中的 **"提交"** 按钮，系统自动校验数据：

1. **校验必填字段**：确保所有必填字段已填写
2. **校验数据格式**：日期格式、数字格式等
3. **校验业务规则**：结束日期不能早于开始日期等

#### 步骤4：确认录入结果

系统返回录入结果卡片：

- ✅ **录入成功**：显示录入的数据摘要
- ❌ **录入失败**：显示错误信息，请修正后重新提交

### 1.4 数据校验规则

| 校验类型 | 说明 | 示例 |
|----------|------|------|
| 必填校验 | 必填字段不能为空 | 项目名称不能为空 |
| 格式校验 | 日期格式：YYYY-MM-DD | 2026-01-15 |
| 范围校验 | 进度范围：0-100 | 进度必须在0-100之间 |
| 枚举校验 | 状态必须为有效值 | 状态必须是"进行中/已完成/暂停" |
| 业务规则 | 逻辑一致性 | 结束日期≥开始日期 |

### 1.5 常见问题

**Q1: 卡片提交后显示"校验失败"？**
A: 请检查必填字段是否填写完整，日期格式是否正确（YYYY-MM-DD），下拉选项是否选择。

**Q2: 如何修改已录入的数据？**
A: 使用"更新项目"指令，或在项目详情中选择"编辑"。

**Q3: 可以批量录入吗？**
A: 批量录入请使用Excel模板导入或飞书在线表格同步功能。

## 2. Excel模板导入操作说明

### 2.1 功能概述

Excel模板导入是通过下载标准模板，在Excel中批量填写数据后上传导入的方式。适合批量录入大量数据。

### 2.2 适用场景

- 批量导入多个项目
- 批量导入大量任务
- 从外部系统迁移数据
- 定期数据批量更新

### 2.3 操作步骤

#### 步骤1：下载Excel模板

在与PM数字员工对话中，输入：

- `下载项目模板` - 获取项目导入模板
- `下载任务模板` - 获取任务导入模板
- `下载风险模板` - 获取风险导入模板

系统返回模板文件下载链接。

#### 步骤2：理解模板结构

每个Excel模板包含两个工作表：

| 工作表 | 内容 |
|--------|------|
| 说明 | 字段说明、填写指南、示例 |
| 数据 | 模板版本、列标题、示例数据行 |

**数据页结构：**

| 行号 | 内容 |
|------|------|
| 第1行 | 模板版本号、更新时间 |
| 第2行 | 列标题（字段显示名） |
| 第3行 | 示例数据（灰色背景，供参考） |
| 第4行+ | 用户填写的数据 |

#### 步骤3：填写数据

按照模板格式填写数据：

1. **必填字段**：列标题红色标记，必须填写
2. **枚举字段**：从下拉列表选择，不要手动输入
3. **日期字段**：格式YYYY-MM-DD
4. **数值字段**：填写纯数字，不要加单位

#### 步骤4：上传导入

将填好的Excel文件发送给PM数字员工：

```
导入项目 [Excel文件]
```

系统自动处理导入。

#### 步骤5：查看导入结果

系统返回导入结果卡片：

| 信息 | 说明 |
|------|------|
| 导入总数 | 本次导入的数据条数 |
| 成功条数 | 成功导入的数据条数 |
| 失败条数 | 校验失败的数据条数 |
| 错误详情 | 失败数据的行号和错误原因 |

### 2.6 导入模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 全量替换 | 清空原有数据，导入新数据 | 数据迁移、初始化 |
| 增量更新 | 按ID匹配，更新已有数据 | 定期数据更新 |
| 仅追加 | 只新增，不更新已有数据 | 新数据补充 |

选择导入模式：

```
导入项目 [Excel文件] --模式=增量更新
```

### 2.7 校验失败处理

如部分数据校验失败：

1. 查看错误详情卡片
2. 修正Excel中的错误数据
3. 重新上传（可选择仅导入失败数据）

### 2.8 常见问题

**Q1: 导入时提示"模板格式不匹配"？**
A: 确保使用的模板是最新版本，列名与模板一致，不要增删列。

**Q2: 日期格式不识别？**
A: 确保日期格式为YYYY-MM-DD（如2026-01-15），不要使用其他格式。

**Q3: 枚举值报错？**
A: 必须从下拉列表选择，不要手动输入自定义值。

**Q4: 如何查看导入历史？**
A: 输入"查看导入日志"查看历史导入记录。

## 3. 飞书在线表格同步操作说明

### 3.1 功能概述

飞书在线表格同步是系统与飞书在线表格双向同步数据的功能。适合多人协作、实时更新场景。

### 3.2 适用场景

- 多人协作编辑数据
- 实时数据更新同步
- 数据可视化展示
- 跨系统集成

### 3.3 操作步骤

#### 步骤1：创建飞书表格

在飞书中创建在线表格，列标题需符合系统字段映射：

| 模块 | 推荐列名 |
|------|----------|
| 项目 | 项目名称、项目编号、开始日期、结束日期、项目经理、状态 |
| 任务 | 任务名称、所属项目、进度、负责人、截止日期、状态 |
| 风险 | 风险名称、风险等级、风险描述、责任人、状态 |

#### 步骤2：绑定表格

与PM数字员工对话：

```
绑定飞书表格 [表格链接] --模块=任务 --项目=[项目ID]
```

系统自动识别表格并创建绑定配置。

#### 步骤3：配置字段映射

系统发送字段映射配置卡片：

1. 确认飞书列名与系统字段对应关系
2. 调整映射配置（如需要）
3. 保存配置

#### 步骤4：设置同步模式

| 同步模式 | 说明 |
|----------|------|
| 单向导入 | 从飞书表格同步到系统 |
| 单向导出 | 从系统同步到飞书表格 |
| 双向同步 | 双向自动同步 |

```
设置同步模式 双向同步
```

#### 步骤5：设置同步频率

| 频率 | 说明 |
|------|------|
| 实时 | 每次修改立即同步 |
| 5分钟 | 每5分钟同步一次 |
| 15分钟 | 每15分钟同步一次 |
| 1小时 | 每小时同步一次 |

```
设置同步频率 15分钟
```

#### 步骤6：执行同步

手动触发同步：

```
执行同步
```

或等待自动同步触发。

#### 步骤7：查看同步状态

系统返回同步状态卡片：

| 信息 | 说明 |
|------|------|
| 同步状态 | 成功/失败/部分成功 |
| 同步时间 | 最近一次同步时间 |
| 同步条数 | 本次同步的数据条数 |
| 冲突条数 | 数据冲突的条数 |

### 3.4 数据冲突处理

当系统数据与飞书表格数据不一致时，产生数据冲突。

#### 冲突解决策略

| 策略 | 说明 |
|------|------|
| 最后写入优先 | 最近修改的数据优先 |
| 飞书优先 | 飞书表格数据优先 |
| 系统优先 | 系统数据优先 |
| 人工确认 | 提示用户选择保留哪个 |

设置冲突解决策略：

```
设置冲突策略 最后写入优先
```

#### 冲突通知

系统发送冲突通知卡片：

1. 显示冲突字段和两边数据值
2. 提供选择按钮（保留飞书/保留系统）
3. 选择后自动解决冲突

### 3.5 常见问题

**Q1: 同步失败？**
A: 检查飞书表格权限，确保机器人有读写权限。检查字段映射是否正确。

**Q2: 数据冲突频繁？**
A: 建议：
- 统一编辑入口，避免两边同时修改
- 设置冲突策略为"最后写入优先"
- 定期检查冲突记录

**Q3: 如何解除绑定？**
A: 输入"解除绑定"解除飞书表格与系统的绑定关系。

**Q4: 同步延迟？**
A: 根据同步频率设置，最长延迟不超过设置的时间间隔。实时同步可能有几秒延迟。

## 5. 常见问题汇总

### 5.1 权限问题

**Q: 某些数据无法查看？**
A: 检查是否有对应项目的查看权限，联系项目管理员授权。

**Q: 无法编辑数据？**
A: 检查是否有编辑权限，部分数据可能需要审批后才能修改。

### 5.2 数据校验问题

**Q: 数据格式要求？**

| 字段类型 | 格式要求 |
|----------|----------|
| 日期 | YYYY-MM-DD（如2026-03-15） |
| 日期时间 | YYYY-MM-DD HH:MM:SS |
| 进度/百分比 | 0-100整数 |
| 金额 | 数字，可带小数点 |
| 状态 | 从下拉列表选择 |

### 5.3 同步问题

**Q: 数据同步延迟？**
A: 根据同步频率设置，最长延迟不超过设置间隔。实时同步可能有几秒延迟。

**Q: 同步冲突如何处理？**
A: 系统按设置的冲突策略自动处理，或提示用户选择。
