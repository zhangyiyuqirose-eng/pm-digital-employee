## 三、发现的Bug与修复

### 3.1 Bug清单

| Bug ID | 问题描述 | 严重程度 | 修复状态 |
|--------|---------|---------|---------|
| BUG-001 | `_standardize_date`日期未补零格式化 | Medium | ✅ 已修复 |
| BUG-002 | `_entity_to_dict`未正确处理None值 | Medium | ✅ 已修复 |
| BUG-003 | Prompt模板JSON示例花括号导致format失败 | High | ✅ 已修复 |

### 3.2 Bug修复详情

#### BUG-001: 日期标准化未补零

**问题描述**: `_standardize_date`方法对中文日期"2024年1月15日"转换为"2024-1-15"而非标准的"2024-01-15"。

**修复方案**: 使用`datetime.strptime`解析后通过`strftime("%Y-%m-%d")`重新格式化确保补零。

**代码修改**:
```python
# 修复前
datetime.strptime(date_str, "%Y-%m-%d")
return date_str

# 修复后
dt = datetime.strptime(date_str, "%Y-%m-%d")
return dt.strftime("%Y-%m-%d")
```

**影响范围**: 所有日期字段提取和数据转换。

#### BUG-002: 实体转字典方法缺陷

**问题描述**: `_entity_to_dict`方法使用`hasattr`检查`to_dict`存在后直接调用，未处理值为None的情况。

**修复方案**: 同时检查`hasattr`和`callable`，并添加`__table__`属性存在检查。

**代码修改**:
```python
# 修复前
if hasattr(entity, "to_dict"):
    return entity.to_dict()
for column in entity.__table__.columns:
    ...

# 修复后
if hasattr(entity, "to_dict") and callable(entity.to_dict):
    return entity.to_dict()
if hasattr(entity, "__table__"):
    for column in entity.__table__.columns:
        value = getattr(entity, column.name, None)
        ...
```

**影响范围**: 数据入库时的实体转换。

#### BUG-003: Prompt模板格式化错误

**问题描述**: `_build_*_prompt`方法使用f-string，但JSON示例中的花括号被Python误解析导致format失败。

**修复方案**: 使用字符串拼接替代f-string，移除JSON示例中的花括号或简化输出要求描述。

**代码修改**:
```python
# 修复前
return f"""...
```json
{
  "report_date": "..."
}
```
"""

# 修复后
return """...
## 输出要求（JSON格式）
请输出包含report_date、week_start等字段的标准JSON。
"""
```

**影响范围**: 所有文档类型的数据提取Prompt构建。
