## 一、项目概述

### 1.1 背景

PM数字员工系统当前仅支持飞书卡片录入方式。为提升数据录入效率和灵活性，需新增Excel模板导入和飞书在线表格实时同步两种数据录入方式，实现项目全生命周期信息的多渠道灵活录入。

### 1.2 目标

- 保留原有飞书卡片录入功能，不做任何修改
- 新增Excel模板导入功能，支持批量数据导入导出
- 新增飞书在线表格双向同步功能
- 三种录入方式数据实时同步、完全一致，无数据冲突

### 1.3 强制约束

1. 原有飞书卡片录入功能**完全保留**，不做任何修改
2. 三种录入方式的数据**实时同步、完全一致**
3. 所有新增功能必须与现有系统的技术栈、数据结构、权限体系、UI风格100%兼容
4. 未收到审批通过指令前，禁止编写任何业务代码

#### 4.2.2 模板设计规范

每个功能模块生成独立Excel模板，包含：

**工作表结构**：
1. **说明工作表**：填写指南、字段说明、示例
2. **数据工作表**：实际数据录入区

**模板格式要求**：
- 模板版本号和最后更新时间（第一行）
- 必填字段用红色`*`标记
- 枚举字段使用下拉选择框
- 数据格式示例行（灰色背景，自动忽略）
- 数据有效性验证（日期格式、数字范围）

## 五、数据同步机制设计

### 5.1 最终一致性模型

采用**最终一致性**模型：
- 所有数据变更先写入核心数据库
- 同步引擎异步更新其他录入方式的数据
- 冲突时以最后修改时间为准

### 5.2 数据流向

```
┌───────────────────────────────────────────────────────────────┐
│                        核心数据库                              │
│                    (PostgreSQL + pgvector)                    │
└───────────────────────┬───────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
┌───────▼───────┐ ┌─────▼─────┐ ┌──────▼──────┐
│ 飞书卡片变更  │ │ Excel导入 │ │ 飞书表格    │
│               │ │           │ │ Webhook事件 │
└───────┬───────┘ └─────┬─────┘ └──────┬──────┘
        │               │               │
        └───────────────┼───────────────┘
                        │
                ┌───────▼───────┐
                │  同步引擎     │
                │ SyncEngine    │
                └───────┬───────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
┌───────▼───────┐ ┌─────▼─────┐ ┌──────▼──────┐
│ 更新飞书表格  │ │ 更新Excel │ │ 更新飞书卡片│
│               │ │ 导出      │ │（可选）     │
└───────────────┘ └───────────┘ └─────────────┘
```

### 5.3 冲突解决策略

| 策略 | 触发条件 | 处理方式 |
|------|----------|----------|
| 最后写入者优先 | 同一记录同时修改 | 以最后修改时间为准 |
| 记录冲突 | 数据内容冲突 | 记录至`data_conflicts`表，供人工确认 |
| 版本标记 | 所有变更 | 记录版本号，支持回滚 |

### 5.4 版本管理机制

每次数据变更自动记录版本：
- 新增记录：version=1，operation=create
- 更新记录：version++，operation=update，记录变更前后数据
- 删除记录：operation=delete，记录删除前数据

支持：
- 查询历史版本
- 回滚到指定版本
- 查看变更轨迹
