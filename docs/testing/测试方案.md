## 一、测试范围

### 1.1 数据管理API测试

| API端点 | 测试内容 |
|---------|----------|
| POST `/api/v1/data/projects` | 创建项目（名称、类型、日期、预算） |
| PUT `/api/v1/data/projects/{id}` | 更新项目信息 |
| POST `/api/v1/data/projects/{id}/tasks` | 创建任务 |
| PUT `/api/v1/data/tasks/{id}` | 更新任务状态 |
| POST `/api/v1/data/projects/{id}/risks` | 创建风险 |
| POST `/api/v1/data/projects/{id}/costs` | 创建成本（预算/实际） |
| POST `/api/v1/data/projects/{id}/milestones` | 创建里程碑 |
| PUT `/api/v1/data/milestones/{id}` | 更新里程碑 |

## 二、测试策略

### 2.1 测试类型

- **单元测试**: Service层、Repository层
- **集成测试**: API端点、数据库交互
- **功能测试**: 飞书卡片处理

### 2.2 测试数据策略

- 使用独立测试数据库
- 测试前清空相关表
- 测试后自动清理数据
- 使用UUID确保数据唯一性

### 2.3 测试环境

- Python 3.12 + pytest 8.x
- PostgreSQL 16 + pgvector
- Docker容器环境
