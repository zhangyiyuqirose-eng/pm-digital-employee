## 二、发现的Bug及修复

### Bug #1: Skill模块导出问题
**问题描述**: `app/skills/__init__.py`中Skill类在函数内导入，模块级别无法访问。

**修复方案**: 将Skill类导入移至模块级别。

### Bug #2: 测试用例日期计算错误
**问题描述**: `test_get_week_range_sunday`使用错误的日期作为周日测试。

**修复方案**: 使用正确的日期（2026-04-19是周日）。

### Bug #3: Token格式验证测试用例长度不足
**问题描述**: 测试用例token长度低于min_length=20的默认要求。

**修复方案**: 使用足够长度的token（≥20字符）。

### Bug #4: Dict Schema验证测试缺少optional_fields
**问题描述**: 测试数据包含unexpected field导致验证失败。

**修复方案**: 在测试中指定optional_fields参数。

### Bug #5: LarkSignatureVerifier构造函数参数错误
**问题描述**: 测试中使用不存在的构造函数参数。

**修复方案**: 使用静态方法。

### Bug #6: IntentResult intent_type必填字段缺失
**问题描述**: IntentResult创建时缺少必填字段intent_type。

**修复方案**: 添加intent_type参数。

### Bug #7: SkillExecutionContext必填字段缺失
**问题描述**: SkillExecutionContext创建时缺少trace_id/chat_id/skill_name必填字段。

**修复方案**: 添加所有必填字段。

### Bug #8: quick_match是async方法
**问题描述**: test_intent_router.py中quick_match调用未使用await。

**修复方案**: 将测试改为async方法并使用await。

### Bug #9: 测试用例Skill不存在
**问题描述**: test_intent_optimization.py中使用cost_estimation等不存在于keyword_mappings的Skill。

**修复方案**: 使用实际存在的Skill名称（cost_monitor等）。

### Bug #10: Orchestrator构造函数参数错误
**问题描述**: test_message_flow.py中Orchestrator构造函数参数不匹配实际实现。

**修复方案**: Orchestrator只接受session参数，修复测试中的Mock设置。

### Bug #11: skill_registry.get_manifest KeyError
**问题描述**: 测试中skill_registry未正确初始化导致找不到skill。

**修复方案**: Mock skill_registry.get_manifest方法返回MockManifest。
